## Description

**The situation:** The sprint board (`src/pages/SprintPlan.tsx`) shows a Kanban board with tasks generated by the sprint-agent. Tasks have title, priority, and status but no RICE scoring (Reach × Impact × Confidence / Effort) or Kano classification (Must-have / Performance / Delight). Founders see all tasks equally weighted with no guidance on which to tackle first.

**Why it matters:** RICE scoring gives founders a data-driven way to prioritize. Kano classification separates hygiene features (must-haves that don't delight) from delighters (features that create wow). Momentum sequencing puts quick wins first to build execution momentum — critical for solo founders who need early wins to stay motivated.

**What already exists:**
- `supabase/functions/sprint-agent/index.ts` — Generates 24 tasks using Gemini Flash, system prompt at L37-47
- `src/pages/SprintPlan.tsx` — Kanban board with drag-and-drop
- `src/hooks/useSprintAgent.ts` — Hook for sprint-agent EF calls
- `agency/prompts/sprint-agent-fragment.md` — Fragment with RICE formula, Kano rules, momentum sequencing

**The build:**
- Import `loadFragment('sprint-agent-fragment')` in sprint-agent
- Append RICE + Kano rules to system prompt
- Add optional fields to task schema: `rice_score` (number), `kano_class` (must_have | performance | delight), `momentum_order` (number)
- Frontend: RICE score badge on each task card
- Frontend: Kano filter tabs above the Kanban board (All | Must-have | Performance | Delight)
- Frontend: "Quick wins first" sort indicator
- Deploy `sprint-agent`

**Example:** Aisha generates a sprint plan. Task "Set up payment processing" gets RICE 84, Kano "Must-have". Task "Add social sharing" gets RICE 42, Kano "Delight". The board shows Must-have tab with 8 high-RICE tasks first, giving Aisha a clear execution sequence.

## Rationale
**Problem:** All sprint tasks look equally important — no prioritization framework.
**Solution:** RICE scoring + Kano classification gives founders objective task ranking.
**Impact:** Founders execute the highest-impact tasks first instead of picking randomly.
