# Core Prompt 03 — Dashboard Screen Design

**Purpose:** Define the luxury, premium dashboard screen with wireframe, content, data, features, and AI agent interactions  
**Focus:** Main dashboard screen specification for core MVP phase  
**Status:** Core Foundation

---

## Screen Purpose

**Daily Execution Overview**

The dashboard is the founder's command center—the first screen they see each day. It provides an intelligent, sophisticated overview of startup health, priorities, and next actions, all presented in a premium, high-end visual experience.

---

### Luxury Premium Aesthetic

**Sophisticated Layout**
- Clean, spacious design with generous whitespace
- Premium typography with clear hierarchy
- Refined color palette with subtle accents

- Professional data visualizations

**Intelligent Information Architecture**
- Most critical information at top

## Wireframe Structure

### Top Section: KPI Bar

**Four Metric Cards in Horizontal Row**
- Monthly Recurring Revenue (MRR)
- Active Users or Customers
- Runway in Months
- Profile Strength Score

**Card Design**
- Clean white background with subtle shadow
- Large, prominent number display
- Clear label below number
- Subtle trend indicator (up/down arrow)
- Optional sparkline chart for trend

**Visual Hierarchy**
- Numbers are primary focus
- Labels are secondary
- Trends are tertiary
- Consistent card sizing
- Generous spacing between cards

### Main Content Area: Three Sections

**Section One: Next Best Action Card**
- Large, prominent card at top
- AI-generated priority action
- Clear, actionable statement
- Single primary button
- Contextual background or icon

**Section Two: Today's Priorities List**
- Top five tasks for today
- Checkbox for completion
- Task title and description
- Priority badge (urgent, high, normal)
- Due date indicator
- Click to expand details

**Section Three: Active Projects Grid**
- Two-column grid of project cards
- Project name and description
- Progress bar with percentage
- Health status badge (on track, at risk, behind)
- Click to view project details

### Right Panel: AI Intelligence

**AI Coach Section**
- Current focus area message
- Personalized guidance text
- Profile completion prompt
- Contextual recommendations

**Risk Radar Section**
- Count of critical risks
- List of top risks with severity
- Brief risk descriptions
- Click to view detailed analysis

**Suggested Actions Section**
- Two to three quick action buttons
- Context-aware suggestions
- One-click execution options
- Clear visual hierarchy

## User Journey

### Morning Routine Flow

**Step 1: Dashboard Load**
- User opens dashboard
- System fetches startup data from Supabase
- System calls RiskAnalyzer agent
- System displays KPIs and content

**Step 2: Review Overview**
- User scans KPI bar for key metrics
- User reviews next best action
- User checks today's priorities
- User views active projects

**Step 3: AI Insights Review**
- User reads AI coach message
- User reviews risk radar
- User considers suggested actions
- User decides on next steps

**Step 4: Take Action**
- User clicks next best action button
- User completes priority task
- User acknowledges AI suggestions
- User navigates to detailed views

**Quick Actions**

- Quick navigation to details
- Dismiss AI suggestions
- Acknowledge risks

**Deep Dives**
- Click KPI card for detailed metrics

## Workflows

### Dashboard Load Workflow

**Trigger:** User navigates to dashboard

**Steps:**
1. Fetch startup data from Supabase
2. Fetch tasks from tasks table
3. Fetch projects from projects table
4. Call RiskAnalyzer agent with data
5. Process AI response
6. Display KPIs, tasks, projects
7. Display AI insights in right panel

**Data Flow:**
- Supabase → Frontend → Edge Function → Gemini API
- Gemini API → Edge Function → Frontend → Right Panel

### Next Best Action Workflow

**Trigger:** User clicks "Take Action" button

**Steps:**
1. Identify action type (task, project, deal)
2. Navigate to appropriate screen
3. Pre-fill form with context
4. User completes action
5. Dashboard updates automatically

**Data Flow:**
- Frontend → Navigation → Detail Screen
- User Action → Supabase → Dashboard Refresh

### Risk Acknowledgment Workflow

**Trigger:** User acknowledges risk in right panel

**Steps:**
1. User clicks "Acknowledge" on risk
2. System marks risk as acknowledged
3. System updates risk display
4. System may suggest mitigation actions
5. Dashboard reflects updated state

**Data Flow:**
- Frontend → Supabase → Risk Status Update
- Supabase → Frontend → UI Update

- AI panel component

**State Management**
- Local state for UI interactions
- React Query cache for data

### Backend Integration

**Supabase Queries**
- Select startup data with org_id filter
- Select tasks with filters and sorting
- Select projects with status filter
- Real-time subscriptions for live updates

**Edge Function Calls**
- Call ai-helper function for RiskAnalyzer
- Pass startup data as context
- Receive structured JSON response
- Parse and display in right panel

**Data Flow Pattern**
- Frontend → Supabase Client → Database
- Frontend → Edge Function Service → Supabase Edge Function
- Edge Function → Gemini API → Structured Response
- Edge Function → Frontend → UI Update
