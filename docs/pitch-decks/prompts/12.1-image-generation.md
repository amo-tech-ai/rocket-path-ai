**Goal:**
Generate clean, investor-ready visuals for EACH slide of a startup pitch deck.

## Global Style Rules

| Rule | Description |
|------|-------------|
| Background | Light, clean, minimal |
| Color Palette | Neutral + 1 accent color (brand or stage-appropriate) |
| Style | Flat, clean, professional |
| Avoid | Cartoonish illustrations, heavy textures, gradients |
| Contrast | High contrast, readable at a distance |
| Aspect Ratio | 16:9 (pitch slide standard) |
| Text in Image | Maximum 3-4 words, leave space for slide copy |

### Visual Language Consistency

All generated visuals must share:
- Same accent color throughout deck
- Consistent icon style (line or filled, not mixed)
- Matching border radius and spacing
- Unified shadow/depth treatment (flat or subtle)

## Model Selection

| Model | Use Case | Quality |
|-------|----------|---------|
| `gemini-3-pro-image-preview` | All pitch deck visuals | Production-grade |

### Why Gemini 3 Pro Image

- Advanced reasoning ("Thinking") for complex visual instructions
- High-fidelity rendering for investor-ready assets
- Better text rendering when labels needed
- Professional asset production quality

### Recommended Workflow

1. **Generate visuals:** Use `gemini-3-pro-image-preview` for all slides
2. **Review with founder:** Get feedback on style direction
3. **Regenerate if needed:** Adjust prompt and regenerate
4. **Insert into editor:** Upload to slide via Prompt 13 editor

---

## Integration Points

### During Deck Generation (Prompt 12)

1. Deck generation completes
2. For each slide type, call image generation
3. Generate slide-appropriate visual
4. Store in Cloudinary / storage bucket
5. Link `image_url` in `pitch_deck_slides` row

### In Editor (Prompt 13)

1. User clicks "Generate Image" in right panel
2. Pass slide type + context to AI
3. Generate visual based on slide content
4. Preview in editor
5. Apply or regenerate
