# Research Reference — Index

Central index for `pm/research/reference/`. Use these lists for **classification** (industries, product types, features, use cases) and **directories** (accelerators, VCs, sources).

---

## How to use this folder

| Need | Use |
|------|-----|
| Classify a startup (industry, product, use case) | **Classification** files below |
| Tag features/capabilities | `03-features.md` |
| Find accelerators or VCs | **Directories** files |
| Run research (URLs, sources) | `100-research-list.md` |

Research prompts (e.g. `pm/research/prompts/02-prompt-startups.md`) reference these for consistent taxonomy.

---

## 1. Classification (taxonomies)

Use when classifying AI startups for reports, tables, or infographics.

| File | Purpose | When to use |
|------|---------|-------------|
| [02-industries.md](./02-industries.md) | **Industries** — Who they serve (B2B, Fintech, Healthcare, etc.) | Primary + sub-industry; optional functional tags |
| [03-ai-product-types.md](./03-ai-product-types.md) | **Product types** — What they build (Vertical AI, Copilot, Infra, etc.) | One primary (optional secondary) per startup |
| [03-features.md](./03-features.md) | **Features** — Capabilities (NLP, RAG, agents, etc.) | Multiple per startup; for tagging and comparison |
| [05-ai-use-cases.md](./05-ai-use-cases.md) | **Use cases** — Jobs to be done (generate leads, automate support, etc.) | 1–3 primary use cases per startup |

**Workflow:** Industry (one) + Product type (one) + Use case (1–3) + Features (many) = full classification row.
