# Project Rules

This directory contains development rules and guidelines that serve as the source of truth for building the StartupAI platform.

## Rules Index

| File | Purpose |
|------|---------|
| [supabase.md](./supabase.md) | Database schema, RLS policies, type usage, auth patterns |

## How to Use

1. **Before starting work**: Review relevant rules
2. **During development**: Follow patterns exactly
3. **Code review**: Verify compliance with rules
4. **New patterns**: Document in appropriate rules file

## Adding Rules

Create new rule files for:
- Component patterns (`components.md`)
- State management (`state.md`)
- API patterns (`api.md`)
- Testing (`testing.md`)
- Styling (`styling.md`)

## Rule Format

Each rule file should include:
- Core principles
- Required practices with code examples
- Anti-patterns to avoid
- Checklists where applicable
